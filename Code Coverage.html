Code Coverage

Code coverage is a key metric used in software testing to measure how much of the source code is executed when test cases run. It helps developers understand which parts of the codebase have been tested and which areas remain untested. By analyzing code coverage reports, teams can identify gaps in their test suites, such as untested functions, conditions, or branches, and work towards improving coverage.

In this session, we will explore how to perform code coverage analysis for both the backend and frontend tests
